RAGFlow 部署总结

📋 环境信息

项目 信息
操作系统 Windows (PowerShell)
Docker Docker Desktop (WSL2 后端)
GPU NVIDIA GeForce RTX 4060 (8GB)
部署模式 CPU 模式（更稳定，更省资源）
RAGFlow 版本 v0.25.0
访问地址 http://localhost:8090
API 端口 http://localhost:9380

🔌 Ollama 模型配置

在 RAGFlow 界面中添加 Ollama 模型时：

配置项 值
基础 URL http://host.docker.internal:11434
API-Key 留空（本地部署）
模型名称 deepseek-r1:8b（对话）或 nomic-embed-text:latest（嵌入）

Ollama 环境配置（如需）

⚠️ 常见问题及解决

问题 解决方案
502 Bad Gateway 等待依赖下载完成（首次启动需 5-15 分钟）
端口冲突 修改 .env 中的端口号
内存不足 增加 Docker Desktop 内存限制到 12GB+
MySQL 连接失败 确保使用 CPU profile：--profile elasticsearch --profile cpu
下载慢 使用国内镜像：registry.cn-hangzhou.aliyuncs.com/infiniflow/ragflow:v0.25.0

✅ 验证部署成功

📝 注意事项

  1. 首次启动需要下载 PyTorch 和 CUDA 依赖（约 2GB），等待 10-15 分钟
  2. CPU 模式比 GPU 模式更稳定，内存占用更低
  3. slim 版本可避免 CUDA 依赖，启动更快
  4. 数据卷包含知识库数据，定期备份
  5. Ollama 地址必须用 host.docker.internal，不能用 localhost
