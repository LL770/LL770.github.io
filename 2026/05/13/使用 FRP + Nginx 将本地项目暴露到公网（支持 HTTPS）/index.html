使用 FRP + Nginx 将本地项目暴露到公网（支持 HTTPS）

适用场景：本地开发机无公网 IP，利用一台有公网 IP 的云服务器，通过 FRP 内网穿透 + Nginx 反向代理，实现通过域名 + HTTPS 访问本地 Web 服务。

环境要求

  • 云服务器：Ubuntu 20.04/22.04/24.04，有公网 IP（示例中记为 你的服务器IP）
  • 本地开发机：Windows / macOS / Linux，运行着 Web 项目（示例端口 3000）
  • 域名：已解析到云服务器 IP（示例中记为 你的域名）

第一步：开放云服务器端口

登录云服务商控制台，在安全组中添加入方向规则（协议 TCP）：

端口 用途 是否必须开放公网
22 SSH 远程连接 是（建议限制来源 IP）
80 HTTP 网站访问 是
443 HTTPS 网站访问 是
7000 FRP 服务端与客户端通信 是
8080 FRP 接收 HTTP 流量 否（仅服务器内部使用）
7500 FRP 仪表板 否（建议用 SSH 隧道访问）

说明：8080 和 7500 不要对公网开放，以减少暴露面。

常见问题排查

现象 可能原因 解决方法
curl 返回 FRP 404 FRP 客户端未连接或未注册代理 检查 frpc 日志，确保 token 一致，且服务端 vhostHTTPPort 正确
公网访问 502 Bad Gateway Nginx 无法连接到 127.0.0.1:8080 检查 frps 是否运行：systemctl status frps
证书错误或无法 HTTPS certbot 未成功或 Nginx 配置错误 重新运行 certbot --nginx
域名无法访问 DNS 解析未生效或安全组未放行 80/443 检查域名 A 记录，检查云安全组规则
本地项目收不到请求 Vite 开发服务器未允许外部域名 在 vite.config.js 中添加 allowedHosts: ['你的域名'] 和 host: '0.0.0.0'
