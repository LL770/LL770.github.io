阿里云服务器初始化与环境搭建指南

本指南记录了从购买阿里云轻量应用服务器到完成基础环境配置（安装 MySQL、Redis、Docker）的全过程，适合新手学习和参考。

1. 服务器选购与配置

  • 产品类型：阿里云轻量应用服务器 或 ECS 经济型e实例
  • 核心配置：2核 2GB
  • 存储：40GB 系统盘
  • 带宽：200Mbps 峰值（轻量） 或 3Mbps 固定（ECS）
  • 操作系统：Ubuntu 22.04 LTS（推荐）

建议：我选择的是 Ubuntu 系统，因为对新手友好，社区支持好，教程丰富。

2. 连接到云服务器

2.1 重置密码

  1. 登录阿里云控制台 → 轻量应用服务器
  2. 点击服务器名称 → 重置密码
  3. 设置符合规则的密码（大小写字母 + 数字）
  4. 等待服务器自动重启（约1-2分钟）

2.2 放行 22 端口

  • 进入控制台 → 防火墙 → 添加规则
  • 协议：TCP，端口：22，来源：0.0.0.0/0
  • tip: 如果SSH连接失败，看一下22端口是否开放

2.3 使用 SSH 连接

方法一：命令行（验证用）

方法二：FinalShell（推荐）

  • 下载并安装 FinalShell
  • 新建连接：主机填公网IP，端口22，用户名root，密码为重置后的密码

七、常见错误及解决方案

错误 1：ERROR 2003 (HY000): Can't connect

原因 解决方案
安全组未开放 3306 端口 阿里云控制台添加入方向规则
MySQL 绑定地址错误 确认 bind-address = 0.0.0.0
服务器防火墙拦截 sudo ufw allow 3306/tcp
MySQL 服务未启动 sudo systemctl start mysql

错误 2：ERROR 1130 (HY000): Host 'xxx' is not allowed

原因 解决方案
root 用户未授权远程访问 执行 CREATE USER 'root'@'%'...
权限未刷新 执行 FLUSH PRIVILEGES;

错误 3：ERROR 1045 (28000): Access denied

原因 解决方案
密码错误 确认密码是否正确
认证插件不兼容 使用 mysql_native_password
