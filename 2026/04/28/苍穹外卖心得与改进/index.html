YApi 部署文档

背景与目的

YApi 是一个开源的接口管理平台（类似 Postman + Swagger 的整合）。但由于官方已停止维护，且最新版 MongoDB 不兼容，部署时需要锁定 MongoDB 4.4 版本。

功能：

  • 接口文档管理
  • Mock 数据
  • 测试集合
  • 团队协作

部署步骤

1. 创建网络

2. 启动 MongoDB 4.4（名称必须为 mongo）

3. 初始化数据库

提示输入时：全部直接按回车，最后输入 y 回车

4. 启动 YApi 服务

5. 验证

项目 内容

账号 admin@admin.com
密码 ymfe.org

常用管理命令

操作 命令
停止 docker stop yapi-server
启动 docker start yapi-server
重启 docker restart yapi-server
日志 docker logs yapi-server
停止并删除 docker stop yapi-server && docker rm yapi-server

注意事项

  1. 必须使用 MongoDB 4.4，最新版不兼容
  2. MongoDB 容器名称必须是 mongo（YApi 脚本硬编码）
  3. 镜像使用阿里云镜像：registry.cn-hangzhou.aliyuncs.com/anoy/yapi（原官方镜像是 jayfong/yapi）
  4. 登录后请立即修改密码

报错

你遇到的核心问题是：存在旧容器/网络，并且没有正确清理。

根据你的日志，问题链条如下：

  1. 容器名冲突 → 名为 mongo 和 yapi-server 的容器已经存在（可能是之前残留的或正在运行的）。
  2. 初始化连接失败 → 因为旧的 mongo 容器可能是用最新版镜像启动的（不兼容），或者根本没有正常运行。

请按以下步骤彻底清理并重新部署（直接复制每行命令执行）：

第一步：彻底清理旧容器和网络

第二步：重新部署（使用一行命令，不要换行）

1. 启动 MongoDB 4.4（新容器）

2. 初始化数据库（直接复制这一整行）

执行后看到提示全部按回车，最后输入 y 确认。看到 初始化管理员账号成功 即可。

3. 启动 YApi 服务（直接复制这一整行）

4. 验证是否成功

如果看到 yapi-server 状态为 Up，立即访问 http://localhost:3000，使用 admin@admin.com / ymfe.org 登录。

为什么会出错？

  1. 旧容器残留：你之前已经创建过 mongo 和 yapi-server 容器（可能是用最新版 MongoDB 运行的），导致新容器无法同名创建。
  2. MongoDB 版本：初始化报错 ENOTFOUND mongo 是因为旧的 MongoDB 容器可能没有正确连接到 yapi-network 或者不是 4.4 版本。

API 设计说明

1 单例资源 vs 多实例资源的 API 设计差异

店铺营业状态（单例资源）

设计特点：

  • 店铺是单一资源，系统中只有一个店铺
  • 不需要额外的 ID 参数
  • status 直接作为路径参数
  • 符合 RESTful 资源导向风格

套餐状态（多实例资源）

设计特点：

  • 套餐是集合资源，系统中有多个套餐
  • 需要通过 id 参数指定具体操作哪个套餐
  • 采用操作式 API 设计
  • 更适合批量操作的扩展

2 为什么设计不同？

特性 店铺状态 套餐状态
资源数量 单一（只有一个店铺） 多个（很多套餐）
是否需要 ID ❌ 不需要 ✅ 需要
设计风格 RESTful 资源导向 操作导向
路径示例 /shop/{status} /setmeal/status/{status}?id=xxx

核心原因： 单例资源可以直接在路径中表达状态变更，而多实例资源需要额外标识具体是哪个实例。
