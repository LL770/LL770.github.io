Maven 高级特性总结

本文档旨在系统化梳理 Maven 的高级功能，帮助应对中大型项目的构建与依赖管理挑战。

1. 分模块设计与开发

  • 定义：将单一 Java 项目按功能或结构拆分为多个独立的子模块。
  • 优势：
    • 易维护：便于大型团队并行开发与管理。
    • 高复用：通用组件（如工具类）可被多个项目独立依赖。
  • 拆分策略：
    1. 按功能模块：如公共组件、商品模块、订单模块。
    2. 按代码分层：如实体层、业务层、控制层。
    3. 混合拆分：功能模块与分层结合。
  • 实践要点：设计先行，先规划模块结构再进行编码。

2. 继承与聚合

2.1 继承 (<parent>)

  • 作用：子模块继承父模块的配置，用于简化依赖配置和统一版本管理。
  • 核心配置：
    • 父工程打包方式必须为 pom。
    • 子工程使用 <parent> 标签指定父工程坐标。

2.2 版本锁定 (<dependencyManagement>)

  • 作用：在父工程中统一声明依赖及版本，仅声明不自动引入。子模块需显式引入坐标但无需写版本号。
  • 区别（面试重点）：
    • <dependencies>：直接引入，子模块自动继承。
    • <dependencyManagement>：版本锁定，子模块需手动引入。

2.3 聚合 (<modules>)

  • 作用：将多个模块组织成一个整体，实现一键构建（如统一编译、打包、安装）。
  • 配置：在聚合工程（通常与父工程合并）的 pom.xml 中添加。
对比项 聚合 继承
核心作用 快速构建项目 简化配置、统一依赖
配置位置 聚合工程中配置子模块列表 子模块中配置父工程坐标
关系感知 父模块知道聚合了谁 父模块不知道谁继承了自己

3. 私服 (Nexus)

  • 定义：架设在局域网内的私有远程仓库，用于解决团队内部依赖共享与外部依赖缓存。
  • 依赖查找顺序：本地仓库 → 私服仓库 → 中央仓库。

3.1 资源配置流程

步骤 配置内容 关键位置
第一步 配置私服访问权限（账号/密码） Maven conf/settings.xml 中的 <servers>
第二步 配置依赖下载地址（镜像/仓库组） Maven conf/settings.xml 中的 <mirrors> 和 <profiles>
第三步 配置项目上传地址 项目父工程 pom.xml 中的 <distributionManagement>
第四步 执行上传指令 运行 Maven 生命周期 deploy
