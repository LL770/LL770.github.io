Git是什么

Git是目前世界上最先进的分布式版本控制系统,用于追踪文件修改、支持多分支并行开发、随时回溯历史版本，并方便多人协作合并代码，且大部分操作在本地完成。

Git工作区和暂存区

  • 工作区：你看到的文件夹，修改/删除文件在这里发生
  • 暂存区：git add 后的临时存放地
  • 仓库：git commit 后的永久存储
    Git跟踪并管理的是修改，而非文件，只有你git add 文件后，git commit 才会提交暂存区的文件到仓库

Git的文件管理

git add [文件名] | .  解释：将指定文件或全部更改的文件添加到仓库

git commit -m "你修改的说明"  解释：把文件提交到仓库并附上说明

git status  解释：查看文件什么状态，未添加、修改、删除

git diff  解释：查看文件与修改前的差异

git log [--pretty=oneline]  解释：查看提交日志 参数是简洁输出

git reset --hard HEAD^  解释：回到上一个版本， HEAD^^ 回到上上一个版本 HEAD~n 回到上n个版本

git reset --hard 1094a   解释：撤回即回到最新版本，输入最新版本的commit id前缀前几位就可以

git reflog  解释：commit id没有找到，查看每一次命令的日志，即可找到对应的commit id

git checkout -- 文件名  解释：把文件在工作区的修改全部撤销，这里有两种情况：
一种是文件自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
一种是r文件已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

git reset HEAD [file]  解释：同样可以把暂存区的修改撤销掉（unstage），重新放回工作区

要删除文件 2种情况 
1 git rm [file]  解释：已经提交 版本库中删除该文件
2 git checkout -- [file]  解释：删错了，恢复到最新版本

git remote add origin git@github.com:LL770/LL770.github.io.git 解释：添加远程库 请更换自己得github用户名并配置SSH Key公钥

git push origin master 解释：把本地master分支的最新修改推送至GitHub

git remote -v  解释：查看远程库信息

git remote rm origin  解释：根据远程库名字origin，删除远程库

git clone 解释：克隆远程仓库
